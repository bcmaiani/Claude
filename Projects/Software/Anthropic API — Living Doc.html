Anthropic API

Living reference — authentication, models, SDKs, and code examples

Last updated: 2026-06-19

The Claude API is a RESTful API at https://api.anthropic.com that provides programmatic access to Claude models and Claude Managed Agents.

There are two broad modes of use:

  • Messages — stateless, direct model calls
  • Managed Agents — stateful, sandboxed, multi-step agent execution (Agents + Sessions + Environments)

Generally Available APIs GA

Beta APIs Beta

Opt in by sending the anthropic-beta request header.

APIEndpointDescription
Files/v1/filesUpload and reuse files across API calls
Skills/v1/skillsCreate custom agent skills

Agents + Sessions + Environments collectively form Claude Managed Agents. Available on the direct API and Claude Platform on AWS only.

C# SDK

Package: dotnet add package Anthropic  ·  Status: Beta v10+

v10+ is the official Anthropic SDK. Versions 3.x and below used a community build (tryAGI.Anthropic) — be aware when reading older tutorials.

Key Features

  • async/await throughout; streaming via IAsyncEnumerable
  • Auto-retry (2 attempts by default) with exponential backoff
  • Default 10-minute timeout — configurable globally or per-call via WithOptions(...)
  • Typed exception hierarchy (AnthropicRateLimitException, Anthropic5xxException, etc.)
  • IChatClient integration for Microsoft.Extensions.AI — plugs into the MCP C# SDK for tool-calling workflows
  • Separate NuGet packages: Anthropic.Bedrock, Anthropic.Vertex, Anthropic.Foundry, Anthropic.Aws
Minimal example — C#
using Anthropic;
using Anthropic.Models.Messages;

AnthropicClient client = new();  // reads ANTHROPIC_API_KEY env var

var message = await client.Messages.Create(new()
{
    Model = Model.ClaudeOpus4_8,
    MaxTokens = 1024,
    Messages = [new() { Role = Role.User, Content = "Hello, Claude" }],
});
Console.WriteLine(message);
Streaming — C#
await foreach (var chunk in client.Messages.CreateStreaming(parameters))
{
    Console.WriteLine(chunk);
}

iOS Options Today

There is no general-purpose Swift/iOS SDK as of June 2026. Three patterns work today:

1 — REST via URLSession

Call https://api.anthropic.com/v1/messages directly with URLSession. Simple, no dependencies.

2 — OpenAI SDK Compatibility

Anthropic exposes an OpenAI-compatible endpoint. Point any OpenAI Swift library at Anthropic's base URL and swap in your Anthropic API key.

3 — Proxy Through Your Backend (Recommended for Production)

Your iOS app calls your own server, which calls the Anthropic API. Keeps the API key off the device entirely.

ClaudeForFoundationModels Requires iOS 27

Repo: github.com/anthropics/ClaudeForFoundationModels  ·  Apache 2.0

A Swift package that conforms Claude to Apple's LanguageModel protocol. Use Claude and Apple's on-device model interchangeably via the same LanguageModelSession API. Requires iOS 27 / macOS 27 / Xcode 27 (all in beta as of June 2026).

Requests go directly from your app to the Claude API — Apple is not in the request path and does not see prompts or responses.

Install

Package.swift
dependencies: [
  .package(url: "https://github.com/anthropics/ClaudeForFoundationModels.git", from: "0.1.0")
]

Or in Xcode: File → Add Package Dependencies… and paste the URL above.

When to Use Claude vs. On-Device

Use SystemLanguageModel (on-device)

  • Fast, private, works offline
  • Lightweight tasks
  • No API key required

Use ClaudeLanguageModel

  • Larger context window needed
  • Frontier reasoning needed
  • Server-side tools (web search, code exec)

Not Supported via Foundation Models Surface

Prompt caching controls (applied automatically), stop sequences, batch processing, Files API, token counting, beta headers.

Quick Start

Swift — ClaudeForFoundationModels
import FoundationModels
import ClaudeForFoundationModels

let model = ClaudeLanguageModel(
    name: .sonnet4_6,
    auth: .apiKey(ProcessInfo.processInfo.environment["ANTHROPIC_API_KEY"] ?? "")
)

let session = LanguageModelSession(model: model)
let response = try await session.respond(to: "Plan a 4-day trip to Buenos Aires.")
print(response.content)

Streaming

Each element is a cumulative snapshot of the response so far, not a delta.

Swift
let stream = session.streamResponse(to: "Summarize today's top science stories.")
for try await partial in stream {
    print(partial.content)
}

Structured Output

Annotate a type with @Generable and request it with generating:. The model returns a value of that type via structured outputs.

Swift
@Generable
struct Trip {
    @Guide(description: "Destination city") var destination: String
    @Guide(description: "Length in days") var days: Int
}

let response = try await session.respond(to: "Plan a trip to Tokyo.", generating: Trip.self)
print(response.content.destination)

Server-Side Tools

Web search, web fetch, and code execution run on Anthropic's infrastructure — nothing is invoked on device. Configure per model with serverTools:.

Swift
let model = ClaudeLanguageModel(
    name: .sonnet4_6,
    auth: auth,
    serverTools: [
        .webSearch(maxUses: 5),
        .codeExecution,
    ]
)

.webSearch and .webFetch also accept domain filters: .unrestricted, .allowing([...]), or .blocking([...]).

Proxy Auth (Production)

Never ship an API key in a binary. Route requests through your own backend; the proxy adds x-api-key server-side.

Swift — production pattern
ClaudeLanguageModel(
    name: .sonnet4_6,
    auth: .proxied(headers: ["X-App-Token": "..."]),
    baseURL: URL(string: "https://api.yourapp.com/claude")!
)
// Your proxy receives standard Messages API requests,
// adds x-api-key, and forwards to https://api.anthropic.com
Coming soon: a production mode using App Attest — no proxy required. Usage billed directly to your Anthropic workspace.

Model Selection & Effort

Use a compiled-in constant
ClaudeLanguageModel(name: .opus4_8, auth: auth)
Pin effort level (overrides framework reasoning hints)
ClaudeLanguageModel(name: .opus4_8, auth: auth, fixedEffort: .xhigh)
// Available: .low  .medium  .high  .xhigh  .max
Custom / unreleased model — declare capabilities explicitly
let model = ClaudeModel(
    id: "claude-experimental-x",
    capabilities: .init(effortLevels: [.low, .high], structuredOutput: true)
)
ClaudeLanguageModel(name: model, auth: auth)

Error Handling

Provider errors that don't map onto LanguageModelError surface as ClaudeError. Pattern-match to drive product flows:

Swift
do {
    let response = try await session.respond(to: prompt)
    print(response.content)
} catch ClaudeError.missingCredential {
    // Prompt for API key
} catch let error as LanguageModelError {
    // Rate limits, guardrails, context length, decoding
} catch {
    // Transport errors
}

Common pattern: catch .rateLimited and fall back to SystemLanguageModel for that turn, or queue for retry.
