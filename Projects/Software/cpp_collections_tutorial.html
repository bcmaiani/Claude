C++ Collections

A hands-on tutorial for intermediate C++ developers

The C++ Standard Library provides a rich set of container types — data structures that manage collections of objects. This tutorial focuses on the four you'll use in almost every real program:

std::vector

Contiguous dynamic array. The default workhorse.

std::set

Sorted collection of unique values. Fast membership test.

std::map

Sorted key→value store. Stable iteration order.

std::unordered_map

Hash table key→value store. Fastest average-case lookup.

All four live in the <vector>, <set>, <map>, and <unordered_map> headers respectively, and are in the std namespace.

Every example in this tutorial compiles cleanly with -std=c++17. A few optional notes call out C++20 additions.

Iterators & Range-for

Before diving into individual containers, it's worth understanding the two patterns you'll use to traverse any of them: range-based for and iterators.

Range-based for (prefer this)

Reads "for each element in the container." Works on all four containers.

#include <vector>
#include <iostream>

int main() {
    std::vector<int> nums = {10, 20, 30};

    // Read-only: capture by const reference (avoids copies)
    for (const auto& n : nums) {
        std::cout << n << "\n";
    }

    // Mutating: capture by reference
    for (auto& n : nums) {
        n *= 2;   // doubles each element in-place
    }
}

Iterators (when you need position)

An iterator is an object that points to an element in a container. You need one when you want to erase during iteration, find an element's position, or use standard algorithms. 

#include <vector>
#include <iostream>

int main() {
    std::vector<int> nums = {10, 20, 30, 40};

    // begin() points to first element, end() points one-past-last
    for (auto it = nums.begin(); it != nums.end(); ++it) {
        std::cout << *it << "\n";   // dereference to get the value
    }

    // Erase the element with value 20 (found at position 1)
    auto it = nums.begin() + 1;   // only valid on vector (random access)
    nums.erase(it);
    // nums is now {10, 30, 40}
}
Use range-for by default. Switch to explicit iterators only when you need to call erase, insert, or pass an iterator to an algorithm.

std::vector

What it is: A resizable array that stores elements contiguously in memory — just like a C array, but it manages its own size. When it runs out of capacity, it allocates a larger block and moves everything over (amortized O(1) push_back).

Declaration & Initialization

#include <vector>

// Empty vector of ints
std::vector<int> v1;

// Five elements, all zero-initialized
std::vector<int> v2(5);

// Five elements, all set to 42
std::vector<int> v3(5, 42);

// Initializer list (C++11)
std::vector<int> v4 = {1, 2, 3, 4, 5};

// Works with any type
std::vector<std::string> words = {"hello", "world"};
std::vector<std::vector<int>> matrix(3, std::vector<int>(4, 0)); // 3x4, all zeros

Modifying Elements

#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {1, 2, 3};

    v.push_back(4);        // append:  {1, 2, 3, 4}
    v.emplace_back(5);     // construct in-place (prefer for objects): {1, 2, 3, 4, 5}

    v.pop_back();          // remove last: {1, 2, 3, 4}

    v.insert(v.begin() + 1, 99);  // insert 99 at index 1: {1, 99, 2, 3, 4}

    v.erase(v.begin() + 1);       // erase index 1: {1, 2, 3, 4}

    v.clear();             // remove all elements, size() → 0 (capacity unchanged)

    std::cout << v.empty() << "\n";   // 1 (true)
}
emplace_back forwards arguments directly to the element's constructor, avoiding a temporary copy. For int or string the difference is trivial, but for heavy objects it matters.

Accessing Elements

#include <vector>
#include <stdexcept>
#include <iostream>

int main() {
    std::vector<int> v = {10, 20, 30, 40, 50};

    // Subscript — fast, NO bounds check (undefined behavior if out of range)
    int x = v[2];          // 30

    // at() — bounds-checked, throws std::out_of_range if bad index
    int y = v.at(2);       // 30

    int front = v.front(); // 10  (first element)
    int back  = v.back();  // 50  (last element)

    std::cout << v.size()     << "\n";  // 5  (number of elements)
    std::cout << v.capacity() << "\n";  // ≥5 (allocated slots)

    // Reserve to avoid repeated reallocations when you know the final size
    v.reserve(100);
    std::cout << v.capacity() << "\n";  // ≥100
    std::cout << v.size()     << "\n";  // still 5
}
If you know roughly how many elements you'll push, call reserve(n) upfront. It avoids repeated reallocations (each of which copies all elements) and keeps your pointers valid.

Iteration

#include <vector>
#include <iostream>
#include <algorithm>

int main() {
    std::vector<int> v = {3, 1, 4, 1, 5, 9, 2};

    // Range-for (simplest)
    for (const auto& x : v) std::cout << x << " ";
    std::cout << "\n";

    // Index-based (when you need the index)
    for (std::size_t i = 0; i < v.size(); ++i) {
        std::cout << i << ":" << v[i] << " ";
    }
    std::cout << "\n";

    // Sort (modifies in place) — O(n log n)
    std::sort(v.begin(), v.end());
    // v is now {1, 1, 2, 3, 4, 5, 9}

    // Binary search (requires sorted vector) — O(log n)
    bool found = std::binary_search(v.begin(), v.end(), 4);  // true
}

std::set

What it is: A sorted collection that holds each value at most once. Implemented as a red-black tree, so all operations are O(log n). Use it when you need fast membership testing and want iteration in sorted order — or when uniqueness is a hard requirement.

Declaration & Initialization

#include <set>
#include <string>

// Empty set of ints
std::set<int> s1;

// From initializer list (duplicates silently dropped)
std::set<int> s2 = {5, 3, 1, 3, 5};  // stores {1, 3, 5}

// From a vector (deduplicates automatically)
std::vector<int> v = {4, 2, 4, 1, 2};
std::set<int> s3(v.begin(), v.end());  // {1, 2, 4}

// Custom comparator: descending order
std::set<int, std::greater<int>> descending = {1, 2, 3};  // iterates 3, 2, 1

Operations

#include <set>
#include <iostream>

int main() {
    std::set<int> s = {1, 3, 5, 7, 9};

    // ── Insert ──
    auto [it, inserted] = s.insert(4);   // structured binding (C++17)
    // inserted == true  → 4 was new
    // *it == 4          → iterator to the inserted element

    s.insert(3);    // no-op: 3 already exists

    // ── Membership test ──
    if (s.count(4)) {                     // count() returns 0 or 1 for set
        std::cout << "4 is in the set\n";
    }
    if (s.find(10) == s.end()) {          // find() returns end() on miss
        std::cout << "10 not found\n";
    }

    // ── Erase ──
    s.erase(3);          // erase by value — no-op if missing
    s.erase(s.find(5));  // erase by iterator

    // ── Size ──
    std::cout << s.size() << "\n";   // current element count

    // ── Range queries (unique to ordered containers) ──
    // lower_bound(x): first element >= x
    // upper_bound(x): first element >  x
    auto lo = s.lower_bound(4);   // iterator to first element ≥ 4
    auto hi = s.upper_bound(7);   // iterator to first element  > 7
    // Iterating [lo, hi) gives elements in [4, 7]
}
Prefer find() over count() when you also need the iterator. Prefer count() (or C++20 contains()) for pure membership checks — they're equally fast but read more clearly.

Iteration

#include <set>
#include <iostream>

int main() {
    std::set<int> s = {5, 2, 8, 1, 9};

    // Always iterates in ascending order
    for (const auto& x : s) {
        std::cout << x << " ";   // prints: 1 2 5 8 9
    }
    std::cout << "\n";

    // Reverse iteration
    for (auto it = s.rbegin(); it != s.rend(); ++it) {
        std::cout << *it << " ";  // prints: 9 8 5 2 1
    }
}
You cannot modify elements through a set iterator — the value is the key, and mutating it would corrupt the sorted order. To change a value, erase it and re-insert.

std::map

What it is: An ordered associative container mapping unique keys to values. Like set, it's a red-black tree — O(log n) operations, sorted key iteration. Use it when you need guaranteed ordering, stable iterators, or range queries over keys.

Declaration & Initialization

#include <map>
#include <string>

// Empty map: string keys, int values
std::map<std::string, int> word_count;

// Initializer list
std::map<std::string, int> scores = {
    {"alice", 95},
    {"bob",   87},
    {"carol", 91}
};

// Any comparable type can be a key
std::map<int, std::string> id_to_name = {
    {1, "apple"},
    {2, "banana"},
    {3, "cherry"}
};

// Custom comparator (keys sorted case-insensitively, for example)
// std::map<std::string, int, MyCmp> custom_map;

Operations

#include <map>
#include <string>
#include <iostream>

int main() {
    std::map<std::string, int> m;

    // ── Insert ──

    // operator[] — inserts with default value if key missing, then assigns
    m["apple"]  = 3;
    m["banana"] = 5;
    m["apple"]  = 7;   // overwrites existing value

    // insert() — only inserts if key is absent; returns pair
    auto [it, ok] = m.insert({"cherry", 2});
    // ok == true  → cherry was new
    m.insert({"banana", 99});   // no-op: banana already exists

    // emplace() — constructs in-place, same semantics as insert
    m.emplace("date", 4);

    // ── Lookup ──

    // operator[] — creates entry with default value if missing (side effect!)
    int n = m["apple"];   // 7

    // at() — throws std::out_of_range if key absent; no insertion
    try {
        int x = m.at("elderberry");   // throws
    } catch (const std::out_of_range&) {
        std::cout << "key not found\n";
    }

    // find() — safest for conditional lookup
    auto search = m.find("banana");
    if (search != m.end()) {
        // search->first == "banana"  (key)
        // search->second == 5       (value)
        std::cout << search->first << ": " << search->second << "\n";
    }

    // count() / contains() (C++20) for existence checks
    if (m.count("apple")) {
        std::cout << "apple is present\n";
    }

    // ── Erase ──
    m.erase("date");              // by key
    m.erase(m.find("cherry"));   // by iterator
}
operator[] on a map always inserts a default-constructed value if the key is missing. Calling m["missing_key"] on a const map won't compile. Use find() or at() for read-only access.

Iteration

#include <map>
#include <string>
#include <iostream>

int main() {
    std::map<std::string, int> m = {{"b", 2}, {"a", 1}, {"c", 3}};

    // Iterates in key-sorted order: a, b, c
    for (const auto& [key, value] : m) {          // structured binding (C++17)
        std::cout << key << " → " << value << "\n";
    }

    // Older style (C++11)
    for (const auto& pair : m) {
        std::cout << pair.first << " → " << pair.second << "\n";
    }

    // Range query: all entries with keys in ["b", "c"]
    auto lo = m.lower_bound("b");
    auto hi = m.upper_bound("c");
    for (auto it = lo; it != hi; ++it) {
        std::cout << it->first << "\n";   // b, c
    }
}

std::unordered_map

What it is: A hash table mapping unique keys to values. Average O(1) lookup, insert, and erase — faster than map for pure key access. No ordering guarantees. Use it when you don't need sorted iteration and throughput matters.

Declaration & Initialization

#include <unordered_map>
#include <string>

// Same syntax as map
std::unordered_map<std::string, int> freq;

// From initializer list
std::unordered_map<std::string, int> capitals = {
    {"France",  33},
    {"Germany", 49},
    {"Japan",   81}
};

// Reserve bucket count upfront to avoid rehashing
// (good when you know approximate number of entries)
std::unordered_map<std::string, int> large_map;
large_map.reserve(1000);    // pre-allocates ~1000 buckets
large_map.max_load_factor(0.7f);  // rehash when 70% full (default 1.0)

Operations

The API is nearly identical to map — the same methods with the same semantics, just without ordering:

#include <unordered_map>
#include <string>
#include <iostream>

int main() {
    std::unordered_map<std::string, int> m;

    // Insert
    m["cat"]   = 1;
    m["dog"]   = 2;
    m.emplace("bird", 3);

    // Lookup — same as map
    auto it = m.find("cat");
    if (it != m.end()) {
        std::cout << it->second << "\n";   // 1
    }

    // Common pattern: count word frequencies
    std::vector<std::string> words = {"the", "cat", "sat", "on", "the", "mat"};
    std::unordered_map<std::string, int> word_freq;
    for (const auto& w : words) {
        word_freq[w]++;   // operator[] default-inits to 0, then increments
    }
    // word_freq["the"] == 2

    // Erase
    m.erase("dog");

    // Iterate (order is arbitrary)
    for (const auto& [key, val] : m) {
        std::cout << key << " → " << val << "\n";
    }
}

Pitfalls & Custom Hash

// ── Pitfall 1: operator[] inserts ──
// Same as map: m["missing"] silently creates an entry with value 0.
// Use find() to check without inserting.

// ── Pitfall 2: iterators invalidated on rehash ──
// Inserting elements can trigger a rehash, which invalidates ALL iterators.
// Don't hold iterators across insertions unless you've called reserve().

// ── Custom hash for user-defined types ──
struct Point { int x, y; };

struct PointHash {
    std::size_t operator()(const Point& p) const {
        // Combine x and y hashes (simple but effective for demo)
        std::size_t hx = std::hash<int>{}(p.x);
        std::size_t hy = std::hash<int>{}(p.y);
        return hx ^ (hy << 1);   // XOR with bit shift to reduce collisions
    }
};

struct PointEqual {
    bool operator()(const Point& a, const Point& b) const {
        return a.x == b.x && a.y == b.y;
    }
};

std::unordered_map<Point, std::string, PointHash, PointEqual> grid;
grid[{0, 0}] = "origin";
grid[{1, 2}] = "point A";
std::hash is specialized for built-in types and std::string out of the box. For any custom struct, you must provide a hash functor (or specialize std::hash).

Which Container to Use?

Scenario Recommended Why
Ordered sequence you'll index or sort vector Cache-friendly, O(1) random access, integrates with std::sort
Deduplicate a list set Uniqueness is built-in; sorted output
Key→value, need sorted iteration or range queries map Ordered, stable iterators, lower_bound/upper_bound
Key→value, pure lookup speed matters unordered_map O(1) average vs O(log n); no ordering overhead
Membership test, no values, unordered unordered_set Like unordered_map but keys only
Small, fixed-size collection vector (or std::array) O(n) linear scan beats tree overhead for n < ~20
When in doubt, start with vector or unordered_map. Profile before switching — the constant factors of hash tables and trees matter more than Big-O at small sizes.

Working with <algorithm>

The <algorithm> header provides generic functions that work on any container via iterators. Here are the ones you'll reach for most often: 

#include <vector>
#include <algorithm>
#include <numeric>
#include <iostream>

int main() {
    std::vector<int> v = {5, 2, 8, 1, 9, 3};

    // Sort ascending
    std::sort(v.begin(), v.end());
    // Sort descending
    std::sort(v.begin(), v.end(), std::greater<int>{});

    // Find first occurrence
    auto it = std::find(v.begin(), v.end(), 8);
    if (it != v.end()) {
        std::cout << "found 8 at index " << (it - v.begin()) << "\n";
    }

    // Count occurrences
    int fives = std::count(v.begin(), v.end(), 5);

    // Find matching elements with a predicate
    auto big = std::find_if(v.begin(), v.end(), [](int x){ return x > 7; });

    // Remove-erase idiom (the standard way to erase matching elements)
    v.erase(std::remove(v.begin(), v.end(), 8), v.end());

    // Sum all elements
    int total = std::accumulate(v.begin(), v.end(), 0);   // initial value = 0

    // Transform: square each element into a new vector
    std::vector<int> squared(v.size());
    std::transform(v.begin(), v.end(), squared.begin(), [](int x){ return x * x; });

    // min and max element
    auto [mn, mx] = std::minmax_element(v.begin(), v.end());
    std::cout << "min=" << *mn << " max=" << *mx << "\n";
}
C++20 introduces ranges (std::ranges::sort(v) instead of std::sort(v.begin(), v.end())), which is cleaner and composes better with views. The semantics are the same; the syntax is shorter.
